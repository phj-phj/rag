## P0 — 检索质量基础

### 1. 低分 chunk 过滤

检索结果 score < 0.4 的不送 LLM，空结果直接返回"未找到"。

### 2. 精确检索替换 IVF_PQ

100+ 文档用暴力搜索，去掉 `.nprobes(20).refineFactor(5)`，相似度不再出现负数。

### 3. Temperature 改为 0

事实性问答不需要随机性，当前 0.3 增加幻觉风险。

### 4. Prompt 去标签化

```
❌ [参考资料1：《xxx》]  →  模型被诱导输出"根据参考资料1..."
✅ 改标签格式 + 移除 prompt 中"参考"字样
```

---

## P1 — 检索增强

### 5. Query 泛化（LLM Rewriting）

口语化问题改写为检索词条，填补语义 gap。一次短 LLM 调用，不改检索链路。

### 6. 文档切片优化

- **NLP 句边界**：用分句模型在自然句号处切断，不截断句子
- **重叠窗口 10-20%**：chunk 尾部拼到下一个 chunk 头部
- **题库适配**：按题号/空行分段，不拆散 Q&A

### 7. 混合检索（向量 + BM25）

向量召回 + LanceDB 全文索引双路融合，关键词搜索命中率补强。

---

## P2 — 用户体验

### 8. 快速/深度双路由

```
问题 → 规则打分（关键词/长度/问号数）
  ├── 简单 → 检索 Top-3 直接拼答案（< 1s，不调 LLM）
  └── 复杂 → 走 MIMO 生成（3-5s）
```

复杂问题可换 `mimo-v2.5`（非 pro）提速，pro 留给深度推理。

### 9. 检索命中检测

```
score ≥ 0.7 → 直接回答
0.4-0.7    → 回答 + "结果可能不完整"
< 0.4      → 不调 LLM，提示换关键词
```

### 10. MMR 多样性去重

Top-20 粗召后逐条精选，优先选不同来源文档，避免 5 条结果全来自同一篇。

## P3 — 锦上添花

- **Reranking**：粗召 20 → SiliconFlow reranker 精排 Top-5
